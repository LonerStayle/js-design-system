# FOUNDATION — 30개 테마 재고도화 공통 계약서

> 확정일: 2026-07-04 · 근거: `docs/fable-upgrade/briefs/` 감사 브리프 23종(theme-01~23) 전수 분석 + 관점 드래프트 3종(debrand / production / hangul) 합성
> 지위: **구현 에이전트가 읽는 유일한 공통 계약서.** 테마별 세부는 각 테마 브리프(§3·§5)가 이 문서를 보완한다.
> 브리프가 아직 없는 테마(24~30)는 이 문서 + 해당 테마 README/tokens.css 주석을 DNA 근거로 삼는다.

## 1. 비전 선언

이 프로젝트는 30개 테마 디자인시스템에서 'AI가 만들었다는 티'를 걷어내고, 실제 프로 디자이너 스튜디오가 출고한 프로덕션 수준으로 끌어올리는 재고도화 작업이다. 걷어낼 것은 차분한 서재 감성 관성, hero+카드그리드 공식, 균질한 리듬, 성의 없는 filler 카피, 분위기 없는 밋밋한 배경이고, 채울 것은 각 테마 DNA를 프로 디자이너처럼 대담하게 밀어붙인 구도·질감·모션·시그니처 모먼트다. 테마의 고유 미학은 단 하나도 훼손하지 않는다 — 오히려 증폭한다. 모든 결과물은 한글 중심 국내 서비스의 실감을 담아야 하며, 접근성·토큰 규율·반응형이라는 프로덕션 품질바를 단 한 항목도 후퇴 없이 통과해야 한다. 완성 판정 질문은 하나다: **"스크린샷 1장으로 이 테마를 다른 29개와 구별할 수 있고, 3초 시연에 '오' 소리가 나는가?"**

## 2. AI 지문 제거 원칙

### 2-0. 대원칙 — 충돌 시 우선순위 (전 문서 공통)

1. **테마 DNA > 이 문서 > 구현자 취향.** DNA(테마 README/tokens.css 주석/CHECKLIST에 명문화된 금지·필수 규칙)와 이 문서가 충돌하면 항상 DNA가 이긴다. 단, 품질 요구(§4)는 포기하는 것이 아니라 **DNA의 어휘로 번역해 충족**한다.

| 상황 | 잘못된 해석 (금지) | 올바른 해석 |
|---|---|---|
| 스위스/다크럭셔리 radius 0~4px 강제 | 포커스 링을 둥글게 | 링도 직각으로 — `outline`+`outline-offset`은 형태 중립 |
| 브루탈리즘/바우하우스 소프트 섀도 금지 | elevation 필요하니 blur 섀도 | 하드 오프셋 섀도·헤어라인·보더로 깊이 표현 |
| 뉴모피즘 보더 금지 | 에러에 빨간 보더 | inset 그림자 심화 + 아이콘 + 텍스트 |
| 미니멀 모노크롬 유채색 0 | 성공=초록, 에러=빨강 | 명도 + 아이콘 + 텍스트 3중 신호 |
| 사이버펑크/블루프린트 영문 시스템 라벨 | `SHEET 20/20`, `ERR::NO_DATA` 번역 | 도면 스탬프·터미널 로그는 DNA. **사용자-facing 본문·aria만 한국어** |
| 미니멀 모노크롬 "노이즈/그레인 금지" | 분위기 배경으로 그레인 | 무기는 명도·스케일·헤어라인뿐 |

2. **인쇄·서적 미학 예외 규칙.** 크림 톤·세리프·문서형 조판이 DNA인 테마(03 에디토리얼, 12 아르데코, 13 다크럭셔리, 15 휘게, 17 보태니컬, 18 미드센추리, 21 리소, 23 손그림, 그리고 20 블루프린트-벨럼 + 잔여 테마 중 24 아카데미아·28 뉴스프린트 추정)에서 그 표현은 **지문이 아니라 자산**이다. 걷어낼 것은 종이·세리프가 아니라 'AI가 만든 문서' 같은 무기력한 실행이다. 목표는 "프로 스튜디오가 만든 인쇄물" — 차이는 목차 조판(점선 리더+페이지 번호), 콜로폰, 도판 넘버링(Tab. XVII), 드롭캡·풀쿼트·마지널리아, 재단선·레지스트레이션 마크, 에디션 넘버링("№ 047/200") 같은 **업계 관례 어휘의 유무**다.
3. **테크·터미널 예외 규칙.** 터미널(07)·인더스트리얼(19)·블루프린트(20)의 영문 대문자 시스템 라벨(`UPTIME`, `SHEET 20/20`, `kernel.panic`)은 명판/도면/로그 픽션 = DNA. 지우지 말고 **세계관 밀도를 균일하게**(약한 페이지를 강한 페이지 수준으로) 끌어올린다. 단 이들 테마도 **읽는 텍스트·aria-label·토스트는 한국어** — "장식/시스템 오브제 = 영문 픽션 허용, 읽는 텍스트 = 한글"의 이중 언어 구조.
4. **지문/DNA 오판 금지 목록** (감사 확정 — 제거하면 안 되는 것): 베이퍼웨이브(08) Cormorant 세리프·전각 라틴·카타카나 장식, 멤피스(10) 크림 캔버스, 다크럭셔리(13) 세리프 디스플레이·아이보리, 스위스(02) Zürich/Helvetica 레퍼런스, 에디토리얼(03) 영문 키커(Culture/Essay), 워드마크(NEO//BRUTAL 등)·코드·클래스명·기술 약어·라틴 학명·CAD 스탬프.
5. **접근성·토큰 규율 절대 회귀 금지.** 각 테마가 이미 달성한 자산(3계층 토큰, 포커스 트랩, reduced-motion, 실측 대비 기록, aria-sort — 브리프 "강점 Top 3")은 보존 계약이다. 새로 넣는 모든 요소도 같은 계약을 따른다.

### 2-1. 서재·문서 감성 관성

- **죽은 세리프 임포트**: `Noto Serif KR` @import 후 폰트 스택 참조 0건이면 화석(19개 테마 적발) — 세리프 DNA 테마를 제외하고 **전량 삭제**. 세리프 DNA 테마는 실제 배선 여부를 확인하고 미배선분만 정리.
- **근거 없는 크림 톤·세리프 헤딩**: 테마 README 팔레트 선언과 대조 — 선언에 없는 웜 크림, 스며든 세리프는 정본 팔레트·타이포로 환원. §2-0-2 예외 테마는 건드리지 않는다.
- **리포트형 얌전한 레이아웃**: 위계 없는 균질 카드, 안전한 2단 분할만 반복하면 세리프가 없어도 '문서 감성'이다 → §3-A로 해소.

### 2-2. 언어·데이터 지문

- **영문 데모 페이지 (P0)**: `lang="en"` + 한글 0줄 페이지 전면 한글화. 범위: `<title>`·aria-label·alt·placeholder·**app.js 내장 문자열**(토스트 기본값, 캘린더 월/요일, "Go to slide", "Notifications")까지 전수. 단순 번역 금지 — **각 테마 index.html이 이미 잡아 놓은 보이스로 번안**(§5-C).
- **서구권 filler 데이터**: "Aria Whitfield", `$1,280.00`, `€`, "Acme", 가짜 로고 스트립("Trusted by 12,000+ teams") → 한국 인명, ₩ 통화, 국내 서비스 맥락(§5-C 규격). 세계관 인명(멤피스 그룹 멤버 등)은 한국어 문맥 안에서 유지 가능.
- **자기참조 dev-tool 카피**: "60+ 컴포넌트 / 0 의존성" 스탯 스트립, "순수 CSS · 프레임워크 없음" 푸터, "v1.0 프로덕션급" 배지 — 제품은 자신을 설명하지 않는다. 삭제하거나 **세계관 수치로 치환**(인더스트리얼=정격 하중·규격, 아르데코="SINCE 1925", 다크럭셔리=크래프트 스펙, 블루프린트=표제란). 푸터는 테마 어법의 콜로폰/명판/판권면으로.
- **raw `.md` 링크**: navbar "문서"→README.md 직링크 전부 제거 또는 데모 내 소개 섹션으로 대체.

### 2-3. 구조 지문 — 공식과 균질 리듬

- **AI 히어로 공식**: `eyebrow 배지 → 대형 h1 → lead(max-width 640px) → CTA 2개 → 4-스탯 스트립`(가운데 정렬이면 가중) → 테마마다 "**포스터/표지/파사드/관제반**"으로 재조판(§3-A, §3-E).
- **균질 카드그리드 진열대**: `repeat(auto-fill, minmax(320px,1fr))` + 전 카드 동일 밀도. 갤러리 자체는 정당한 목적 — **진열을 없애는 게 아니라 위계를 만든다.** 대표 컴포넌트 2×2 스팬 히어로 셀, 소형은 조밀한 멀티컬럼, 최소 한 요소는 그리드 침범(오버랩/rotate/bleed). 테마 어법 재해석: 잡지=목차 조판, 도면=등록 대장, 몬드리안 구획, 틸링 윈도우.
- **균질 섹션 리듬**: 전 섹션 동일 padding + section-head 복붙 → **의도된 강약**(넓게→빽빽→초대형 여백+전폭 룰). 섹션 번호를 초대형 워터마크 숫자/챕터 표지로 승격. 풀블리드 브레이크 섹션(색면 침수, 반전 섹션) 페이지당 1개 이상.

### 2-4. 성의 없는 실행 지문

- **이모지 아트워크/아이콘**: 상품 갤러리·cmdk·빈 상태의 `📊🗂️✉️` → 시스템의 기존 일러스트 어휘로 교체(클레이 clay-char, 휘게 코티지 라인아트, 아르데코 시계 다이얼). 카와이처럼 이모지가 무드에 맞아도 **자체 SVG 모티프**가 정답(OS 렌더 편차 제거 + 브랜드 각인).
- **동일 장식 복붙**: 코너 블롭·radial glow 레시피를 페이지마다 좌표만 바꿔 반복 → **페이지별 씬 변주**(§6-2).
- **인라인 스타일 폭주**: 페이지당 50개+ → 3회 이상 반복 패턴은 클래스로 승격 의무. **인라인 `grid-template-columns` 전면 금지**(§4-6).
- **죽은 자산**: 정의만 있고 사용 0건인 토큰/키프레임/유틸(`--noise-opacity`, `clay-wiggle`, `--shadow-yellow`, `--z-cursor` 등) — **삭제가 아니라 점화가 기본.** 이 죽은 자산들이 대부분 그 테마 대담화의 정답이다(§3). 점화 불가능하면 삭제. 방치 금지.

### 2-5. 모션·팔레트 지문

- **범용 fade-up**: `[data-reveal] { opacity:0; translateY(…) }` + 등차 스태거가 전 테마 동일 → **테마 물리로 교체**(§3-C). fade-up은 폴백으로만.
- **클리셰 팔레트**: 보라 그라데이션 배경클립 텍스트, Tailwind 원값 유출(`#6366f1`), GitHub Dark 코드블록 차용 → 자기 램프로 환원. DNA 금지 기법의 밀수(멤피스 skeleton의 gradient 샤인 등)는 테마 어법으로 교체(candy-stripe 등).

## 3. 대담화 원칙 — 프로 디자이너의 4대 무기

> 공통 문법: 전부 **토큰 경유**로 구현, 신규 값은 시맨틱 레이어에 등록. 모든 모션에 `prefers-reduced-motion` 가드, 모든 장식 레이어에 `aria-hidden="true"` + `pointer-events:none`.

### 3-A. 대담한 구도 — 페이지당 최소 1곳, 그리드가 "깨진" 지점

깨는 방식은 테마 어법을 따른다.

```css
/* 1) 초대형 타이포 + bleed/crop */
.hero-display {
  font-size: clamp(4rem, 14vw, 12rem);
  line-height: 0.95;                          /* 한글이면 §5-A 보정값 */
  margin-inline: calc(var(--space-6) * -1);   /* 컨테이너 침범 */
}
.hero { overflow: clip; }                     /* body overflow-x:hidden 전역핵 금지 */

/* 2) 오버랩(z-교차) — 이동 모션 금지 테마(스위스)에서도 정적 겹침은 합법 */
.hero-title { z-index: 1; margin-top: calc(var(--space-10) * -1); }
.break-out  { margin-bottom: calc(var(--space-12) * -1); position: relative; z-index: 2; }

/* 3) 비대칭 벤토 — 균등 4칸 stat 그리드 금지, 핵심 지표 하나를 2×2 히어로 셀로 */
.bento { display: grid; grid-template-columns: repeat(4, 1fr);
  grid-template-areas: "hero hero a b" "hero hero c d"; }
.stat--hero { grid-area: hero; }

/* 4) 대각선·회전 — 섹션 절단, 마퀴/테이프 스트립, 스티커 회전 */
.section--cut { clip-path: polygon(0 0, 100% 4rem, 100% 100%, 0 calc(100% - 4rem)); }
.sticker { rotate: var(--tilt-ccw, -2deg); }

/* 5) 세로쓰기·조판 변주 — 한글 테마의 무기 */
.vertical-label { writing-mode: vertical-rl; }
```

**금지선**: 구도 파괴가 가독성·키보드 순서·반응형을 깨면 안 된다. 오버랩·블리드는 **360px 낙하 형태까지 함께 설계**(§4-6-4).

### 3-B. DNA에 맞는 분위기 배경 — 발명하지 말고 점화하라

"밋밋한 단색 배경 = AI 지문". 무엇을 깔지는 DNA가 결정하며, 씨앗(죽은 토큰)이 이미 각 테마에 있다.

- **노이즈/그레인** (터미널·다크럭셔리·미드센추리·리소·휘게): SVG feTurbulence data-URI, 초저알파, `--grain-opacity` 노브.
- **구조적 질감** (그리드 금욕 테마의 합법 어휘): 재단선/크로스헤어/치수선/모눈 — 도트·텍스처 금지 테마(스위스·모노크롬)에서도 1px 선은 DNA 정합.
- **깊이·빛**: 테마 광학으로 — 오로라 스크롤 시프트(04), 촛불 halo(15), 선버스트(12), 별사탕 밤하늘(16 다크), 페이지별 색 온도(07).

```css
/* 섹션/페이지 스코프 분위기 노브 — 씬 변주의 기반 */
[data-scene="midnight"] { --fx-sun: 0; --fx-star-opacity: .8; }
[data-scene="dawn"]     { --fx-sun: 1; --fx-grid-floor: .3; }
```

**배경 레이어 계약**: `pointer-events:none` + `aria-hidden` + **콘텐츠 대비 재검증**(그레인이 텍스트 위에 깔리면 실측 대비 하락 — 리소 감사 확인). 라이트/다크 각각 오버라이드 필수(라이트에서 소멸하는 흰 라인 텍스처 사례 있음).

### 3-C. 시그니처 모션 시스템 — 테마당 1문법

테마당 **시그니처 모션 문법 1개**를 정하고, 로드 스태거·스크롤 리빌·호버가 전부 그 문법의 변주가 되게 한다.

```css
/* 예: 스탬프(브루탈/코믹/인더스트리얼) — 위에서 '쾅' */
@keyframes stamp-in { 0% { opacity:0; transform: scale(1.18) rotate(-2deg); }
```

- `prefers-reduced-motion: reduce` → 정지 또는 즉시 완성 상태(숨김 방치 금지). 파티클·버스트류는 **생성 자체를 스킵**.

### 3-D. 시그니처 모먼트 설계법

## 4. 프로덕션 품질바

품질 규칙과 DNA가 충돌하는 것처럼 보이면 §2-0-1 표대로 **표현 방법을 테마 언어로 번역**한다. 품질 자체는 협상 불가.

### 4-1. SVG 색상 규칙 — presentation attribute의 var() 전면 금지 (P0 렌더 버그)

CSS 커스텀 프로퍼티는 SVG presentation attribute에서 해석되지 않는다 (16개 테마, 최대 64건/테마 적발). 결과: `stroke` → 선 소실, `fill` → 검정 뭉개짐, `stop-color` → 그라데이션 붕괴.

```html
<!-- ❌ 전부 금지 -->
<path stroke="var(--color-primary)" />
<rect fill="var(--color-surface)" />
<stop stop-color="var(--gold-500)" />
<text font-family="var(--font-mono)" fill="var(--color-text-subtle)">라벨</text>

<!-- ✅ 패턴 A: style 속성 — 그라데이션 <stop>·일회성 일러스트 -->
<path style="stroke:var(--color-primary)" />
<stop style="stop-color:var(--gold-500)" stop-opacity="0.22" />

<!-- ✅ 패턴 B: currentColor + 부모 color — 단색 아이콘의 정석, 다크모드 자동 추종 -->
<span class="icon-star"><svg><path stroke="currentColor" fill="none"/></svg></span>
<style>.icon-star { color: var(--color-warning); }</style>

<!-- ✅ 패턴 C: CSS 클래스 — 차트·계열색·다요소 일러스트의 정석 -->
<polyline class="chart-line chart-line--primary" points="…"/>
<style>.chart-line--primary { stroke: var(--color-primary); }</style>
```

**오탐 주의 (수정 금지)**: CSS 파일 내부 `stroke: var(--x)`(정상), JS의 `el.style.fill = 'var(--x)'`(정상), `data-fill` 속성을 JS가 style로 소비하는 패턴(정상), 상품 사진 성격의 하드코딩 hex SVG(의도된 콘텐츠 색 — 주석만 부여).

**검증** (결과 0건이어야 통과):
```bash
grep -rn 'stroke="var(\|fill="var(\|stop-color="var(\|color="var(\|font-family="var(' \
  --include="*.html" design-systems/theme-XX-*/
```

### 4-2. 토큰 규율 — 3계층 계약

```
tokens.css (원시 램프) → semantic.css (역할 토큰) → components/*.css (시맨틱만 소비)
```

- 컴포넌트·페이지는 시맨틱 토큰만 참조. 원시 램프 직접 참조는 semantic.css 정의부에서만.
- **대담화 신규 스타일도 예외 없음** — 오버랩 히어로·시그니처 모션의 색·간격·이징도 전부 토큰 경유. 필요한 값이 없으면 토큰을 **신설**하고 주석으로 근거를 남긴다(`--glass-sweep`, `--grain-opacity` 식으로 시스템 전파). 즉흥 hex 금지.
- **하드코딩 금지**: 페이지/컴포넌트 hex·rgba 직접 기입, 인라인 `style="color:#fff"` 땜질, 팔레트 외 색(존재 자체가 결함), data-URI 내부 고정 색(다크에서 죽는 select 캐럿 등). 허용 예외는 `/* intentional: mask color */` 주석 의무.
- **유령 토큰 금지**: `var(--dur-2, .18s)` 같은 미정의 토큰+폴백 연명 패턴은 실존 토큰으로 교체. 정의만 되고 미사용인 토큰은 ① 대담화에서 점화 ② 삭제 — 방치 금지.
- **다크모드 재정의 완전성**: 라이트에서 소비되는 모든 토큰은 다크 블록에서 유효값을 가져야 한다. 램프 일부만 재정의(잔존 심색), 구조 보더 `var(--black)` 하드코딩(다크에서 윤곽 소실 71곳 사례), 라이트 고정 패턴 노출 — 전부 결함. 정적 검증 + **육안 확인**(정적 검증만 통과한 버그 사례 있음). 패턴/텍스처/캐럿/차트 defs/JS 하드코딩 색 배열까지 전수. 다크는 "색 반전"이 아니라 테마 재해석 서사(칠판, 야간 인쇄소, 앰버 터미널)를 따르되, **재정의 누락은 서사가 아니라 결함**이다.

### 4-3. 인터랙션 상태 완전성

| 컴포넌트 | 필수 상태 |
|---|---|
| 버튼(**전 변형**) | default · hover · **focus-visible** · active · disabled · loading |
| 입력류 | default · hover · focus · **invalid(+에러 메시지)** · disabled · placeholder |
| 선택 컨트롤 | + checked/on, 키보드 조작 |
| 리스트/테이블 행 | hover · selected · focus-visible |
| 데이터 뷰 | **empty** · loading(skeleton) · error |
| 오버레이 | open · **closed(불투명·비간섭)** · empty 결과 |

- "전 변형" 의무: primary만 완비하고 danger/soft는 hover만 있는 사례 재발 금지.
- **상태는 "정의"가 아니라 "시연"까지**: CSS에 정의된 모든 상태는 데모 화면 어딘가에 **실사용 맥락**으로 최소 1회 노출(§6-3). 정의만 있으면 미완성.

**focus-visible 계약**:
```css
:where(a, button, input, select, textarea, [tabindex]):focus-visible {
  outline: 2px solid var(--color-ring);
  outline-offset: 2px;
}
:focus:not(:focus-visible) { outline: none; }   /* 전역 outline:none 단독 금지 */
```
- **Cascade 함정**: 링을 `box-shadow`로 구현하면 뒤에 로드되는 컴포넌트 CSS의 동일 채널이 링을 덮는다 → 그림자 컴포넌트는 자체 `:focus-visible`에서 `box-shadow: var(--btn-shadow), 0 0 0 2px var(--color-surface), 0 0 0 4px var(--color-ring)` 합성, 또는 링을 `outline` 기반으로(채널 분리).
- 링 형태는 대상 형태를 따른다(`border-radius: inherit`) — pill 버튼에 각진 링 금지. 링 색·질감은 테마 DNA로(네온, 골드 헤어라인, 잉크 2px) — **존재 자체는 협상 불가**.

**닫힌 오버레이의 비간섭 계약** (투명 클릭 가로채기·탭 잔존 버그 예방):
```css
.modal:not(.is-open) {
  opacity: 0;
  visibility: hidden;        /* 전환 시 transition: visibility 0s .2s */
  pointer-events: none;      /* 또는 JS로 inert 부여 */
}
```
오버레이 공통 의무: 열림 시 포커스 이동 → Tab 순환 트랩 → ESC → 닫힘 시 트리거로 포커스 복원 → 스크림(바깥 클릭 닫기). **드로어·오프캔버스도 모달과 동급** — 모달만 완비하고 사이드바는 전무인 불일치 금지. 숨김 오버레이 안 `autofocus` 속성 금지(로드 시 스크롤 점프).

### 4-4. 접근성 — WCAG AA · ARIA · 키보드

**대비**: 본문 ≥ 4.5:1 · 대형 텍스트(24px+/18.66px+ bold) ≥ 3:1 · 비텍스트 UI(보더·아이콘·포커스 링) ≥ 3:1 · placeholder도 텍스트로 취급 ≥ 4.5:1 · 12px 미만 초소형 라벨에 subtle 토큰 금지.
- 그라데이션·패턴·유리 배경 위 텍스트는 **가장 불리한 지점 기준** 판정 — 미달 위험 시 솔리드 표면으로 이동 또는 가독 스크림.
- 채도 높은 브랜드색을 텍스트로 쓸 땐 **배경용/텍스트용 토큰 분리**(`--pen-*-dark` 패턴 — 한 단계 어두운 전용 토큰).
- 측정값을 CHECKLIST에 **수치로 기록**, 변경 시 재실측, 라이트/다크 양쪽.

**ARIA 표준 교정 패턴** (감사 최다 오용):
```html
<!-- 세그먼트 (10개+ 테마 오용): plain button에 aria-selected는 무효 ARIA -->
<div class="segmented" role="radiogroup" aria-label="기간 선택">
  <button role="radio" aria-checked="true">일</button>
  <button role="radio" aria-checked="false">주</button>
</div>
<!-- 토글 그룹: role 없이 aria-pressed 단일화 · 진짜 탭(패널 전환)만 tablist 완전체 -->
```
- JS 동기화 필수: 클릭·방향키 모두에서 `aria-checked`/`aria-pressed` 갱신(클래스만 토글 금지).
- 별점: 입력용 = radiogroup+radio+방향키(`role="slider"` 선언 후 키보드 미구현은 거짓 ARIA). 표시용 = **인터랙티브 금지**(span). aria-label=시각=data 3자 일치 — half-star는 클립/마스크로 실제 렌더.
- cmdk: `role="listbox"`+`role="option"`+`aria-activedescendant`, 빈 결과 요소 실존+`aria-live`.
- 구조 오류 금지: `role="img"` 안 인터랙티브 버튼, `aria-hidden` 안 포커스 가능 요소, role 없는 span의 aria-label, deprecated `aria-grabbed`, 중복 class/style 속성.
- 라이브 리전: 토스트 = `role="status"`+`aria-live="polite"`(에러는 `role="alert"`). `role="region"`만으로는 안 읽힌다.
- 차트: `role="img"`+데이터 요약 aria-label("3월 38백만원에서 6월 82백만원으로 상승") 또는 visually-hidden 표. 진행률 = `role="progressbar"`+`aria-valuenow/min/max`.

**키보드 — 마우스 전용 기능 제로**:
| 패턴 | 표준 해법 |
|---|---|
| 테이블 정렬 th | `<th scope="col" aria-sort="…"><button class="th-sort">컬럼명</button></th>` |
| 칸반 DnD | 카드에 대안 버튼 "← 이전 열로 / 다음 열로 →" + 라이브 리전 안내 |
| 카드형 링크(cursor:pointer div) | `<a>` 래핑 또는 button화 |
| 드롭존 `role="button"` | Enter/Space keydown 의무 |
| `role="menu"` | 방향키 구현 또는 role 완화 |
| 캐러셀 | 방향키 + dots `aria-current`, 썸네일은 커스텀 이벤트 구독 |

- 스킵 링크 전 페이지, **한국어**("본문으로 건너뛰기"). `href="#"` 더미 링크 금지.
- 스크린리더 문구는 테마 DNA와 무관하게 **무조건 한국어**.

### 4-5. 다크모드 전환 — FOUC 방지 (6개 테마 결함)

전 HTML `<head>` 최상단에 동기 스니펫:
```html
<script>
(function () {
  try {
    var t = localStorage.getItem("theme");
    if (!t) t = matchMedia("(prefers-color-scheme: dark)").matches ? "dark" : "light";
    document.documentElement.setAttribute("data-theme", t);
  } catch (e) {}
})();
</script>
```
- 도입 시 HTML의 `data-theme` 하드코딩 속성 제거(하드코딩이 남으면 `:root:not([data-theme])` CSS 폴백이 영원히 죽은 코드).
- JS 문자열이 HTML 한글 라벨을 영문으로 덮어쓰는 한/영 혼종 금지 — 라벨 소스는 한 곳.

### 4-6. 반응형 — 360px 모바일 우선

1. **검증 뷰포트 계약: 360 / 768 / 1024 / 1440px × 라이트/다크** 육안 확인. "All responsive ✅" 자가 주장은 확인 근거 없이 기록 금지.
2. **내비게이션 수단이 0이 되는 뷰포트 금지** (9개 테마 붕괴 지점): 죽은 버거 버튼 금지(연결 대상 없으면 만들지 않는다), CSS가 참조하는 컨트롤은 전 페이지 마크업에 실존, 숨기는 미디어쿼리와 토글을 보이는 미디어쿼리는 같은 브레이크포인트 값 공유(데드존 금지), 3-pane 패널 축소 시 전환 경로 제공(목록↔읽기 스택, 드로어 승격), 오프캔버스 열림 시 §4-3 오버레이 계약 동일 적용.
3. **레이아웃 붕괴 방지**:
```html
<!-- ❌ 인라인 grid — 미디어쿼리가 못 덮어 모바일 압착 -->
<div style="grid-template-columns:1.6fr 1fr">
<!-- ✅ 클래스 + 브레이크포인트 -->
<div class="grid-main-side">
<style>
.grid-main-side { display:grid; grid-template-columns: 1.6fr 1fr; gap: var(--space-6); }
@media (max-width: 820px) { .grid-main-side { grid-template-columns: 1fr; } }
</style>
```
- 공용 `.grid-2/3/4` 유틸에 축소 브레이크포인트 내장 의무. 고정폭 사이드바+`1fr`은 축소 시 드로어 전환. 전고 레이아웃은 `100dvh`(`100vh` 금지). `body { overflow-x: hidden }`은 버그 은폐 수단 — 개별 요소에서 처리. 모바일에서 시그니처 비주얼 통째 `display:none` 금지(축소·재배치 원칙). transform scale 축소는 레이아웃 박스를 안 줄인다(유령 여백 주의).
4. **대담화와 반응형**: 오버랩·비대칭·블리드 연출은 360px 낙하 형태까지 함께 설계 — 겹침은 순차 스택으로, 초대형 타이포는 `clamp()`로, 절대배치 장식은 숨김이 아니라 재배치.

### 4-7. 폰트 로딩 성능

- `@import`/`<link>`의 모든 패밀리·웨이트는 **사용처 실존** 의무 — 미사용 Noto Serif KR 유령 임포트(19개 테마) 등 전량 정리(§2-1).
- @import 다중 분산(tokens.css+base.css 렌더 블로킹 직렬) 금지 — 한 곳 통합 또는 HTML `<link rel="preconnect">`+단일 `<link>` 승격(권장). `display=swap` 유지. 웨이트는 사용분만. 로딩 경로가 CSS/HTML 두 갈래로 갈라진 상태 금지.

### 4-8. 마크업·코드 위생

- 깨진 참조 0건: 존재하지 않는 파일 href·앵커, cross-SVG defs 의존(자체 defs 소유 원칙).
- HTML 유효성: 중복 class/style 속성(파서가 첫 값만 채택), void 요소 내부 콘텐츠, button의 href 등 무효 마크업 금지.
- 이벤트 바인딩 일원화: 인라인 `onclick` 금지 — data-attribute 위임 아키텍처 준수.
- JS 위생: 텍스트 삽입은 `textContent`(innerHTML 금지), 포커스 트랩 핸들러는 요소별 WeakMap(전역 단일 핸들러 누적 버그), 스크림 탐색을 DOM 형제 순서에 의존 금지.
- 성능: 전면 고정 텍스처의 상시 애니메이션은 `will-change: transform` 또는 모바일 정지 분기. `drop-shadow`는 spread 인자 미지원(무효 선언 주의).
- 데모 데이터 정합: 수치·좌표 복붙 금지, 범례=데이터 의미 일치, 7일 데이터를 8칸에 끼우기 금지.
- file:// 더블클릭 실행에서 콘솔 에러 0, 깨진 리소스 0.

## 5. 한글 조판 + 카피 보이스

### 5-A. 한글 조판 기저 규격

**1) 줄바꿈 계약** — 전 테마 `keep-all` 0건이었다. 가장 값싸고 효과 큰 수리:
```css
body {
  word-break: keep-all;          /* 어절 단위 줄바꿈 */
  overflow-wrap: break-word;     /* 긴 URL·영문 토큰 안전망 */
  line-break: strict;
}
h1, h2, h3, .card__title, .hero__title { text-wrap: balance; }
p { text-wrap: pretty; }
/* 좁은 컨테이너(셀·칩·배지)는 말줄임이 정답 — keep-all 오버플로 방지 */
.table td, .chip, .badge { white-space: nowrap; overflow: hidden; text-overflow: ellipsis; }
```
초대형 디스플레이 타이포는 줄바꿈을 브라우저에 맡기지 않는다 — 카피를 **의도한 줄 단위로 `<span>` 분절**:
```html
<h1 class="display"><span>부수고. 만들고.</span><span>출시하라.</span></h1>
```

**2) 행간·자간 이원화** — 라틴 수치를 한글에 이월하지 말 것. 토큰 신설 + `:lang(ko)` 분기:

| 역할 | 라틴 | 한글 기저 |
|---|---|---|
| 본문 단락 | 1.5~1.6 | **1.65~1.75** |
| UI 텍스트 | 1.2~1.4 | **1.4~1.5** |
| 헤딩(다행) | 1.1~1.2 | **1.25~1.35** |
| 디스플레이 | 0.9~1.05 | **1.05~1.2** (`line-height:0.95` 한글 직행 금지 — 겹침·잘림) |
| 와이드 라벨 자간 | 0.1~0.34em | **0.05~0.1em**, 극단 트래킹도 한글 상한 **0.15~0.2em** |
| 디스플레이 자간 | -0.02~-0.04em | **-0.02em 상한** (Black Han Sans·Do Hyeon·Jua 등 고밀도 무드폰트는 **0**) |

`--leading-none(1.0)`은 한 줄짜리 워드마크·스탯 숫자에만. 넓은 자간 중앙정렬 한글 라벨은 마지막 글자 뒤 자간만큼 `margin-right: calc(var(--tracking-caps-ko) * -1)` 시각 보정.

**3) 폰트 스택 순서 (강제)** — `system-ui`가 한글 무드 폰트보다 앞이면 macOS에서 무드 폰트 영원히 미적용(8개+ 테마 적발):
```css
/* 순서: ① 라틴 무드 → ② 한글 무드 → ③ 한글 안전망 → ④ 시스템 제네릭 */
--font-display: "Space Grotesk", "Black Han Sans", "Noto Sans KR", system-ui, sans-serif;
```
라틴·숫자는 여전히 ①이 선점하므로 부작용 없음. **숫자가 라틴 폰트로 렌더되는 것이 혼용 리듬의 뼈대**(tabular 피처도 라틴 폰트에서 온다).

**4) 웨이트 정합 — faux-bold 근절**: 헤딩이 쓰는 웨이트(700/800/900)를 한글 폰트가 실보유·실로드하는지 대조. 미보유 시 (a) 폴백 승격 (b) 웨이트 추가 로드 (c) 단일 웨이트 무드폰트(Jua·Do Hyeon·Black Han Sans 400)는:
```css
:lang(ko) :is(h1, h2, .display) {
  font-synthesis-weight: none;
  font-weight: 400;              /* 위계는 크기·자간으로 보정 */
}
```

**5) 이탤릭·스몰캡스 무효 처리**: 한글엔 이탤릭이 없다(faux-oblique 왜곡). `small-caps`·`uppercase`는 한글에 무효.
```css
:lang(ko) :is(em, i, cite) { font-style: normal; font-weight: var(--weight-semibold); }
:lang(ko) .label { font-variant: normal; font-size: var(--text-xs);
  letter-spacing: var(--tracking-caps-ko); }   /* 크기 축소+자간으로 라틴 스몰캡스와 위계 맞춤 */
```
라틴 학명·인용구는 라틴 문자로 유지해 이탤릭을 살린다. 명조 테마는 서체 전환(본문 명조→강조 바탕)으로 목소리 구분 가능.

**6) 숫자 피처**: 데이터(테이블·스탯·가격·카운터)는 `font-variant-numeric: tabular-nums lining-nums`. 인쇄 DNA 테마의 old-style 본문 숫자 관습은 유지. 전제: 숫자가 라틴 폰트에서 렌더되도록 스택 순서(3) 선행.

**7) 표준 이식 스니펫** — 각 테마 base.css 하단, 테마 토큰만 참조:
```css
/* === Hangul refinement layer — 전제: <html lang="ko"> + 스택 순서 교정 === */
:lang(ko) { word-break: keep-all; overflow-wrap: break-word; line-break: strict; }
:lang(ko) :is(h1, h2, h3) { line-height: var(--leading-heading-ko, 1.3); }
:lang(ko) .display        { line-height: var(--leading-display-ko, 1.12); }
:lang(ko) :is(.eyebrow, .label, .btn--caps) {
  letter-spacing: var(--tracking-caps-ko, 0.08em);
  text-transform: none; font-variant: normal;
}
:lang(ko) :is(em, i, cite) { font-style: normal; font-weight: var(--weight-semibold, 600); }
:lang(ko) :is(.table td, .stat__value, .price) { font-variant-numeric: tabular-nums lining-nums; }
```
이 레이어는 기저값 — 인쇄 DNA 테마의 자기 조판 관습이 오버라이드한다(§2-0 우선순위).

**8) 페어링 보존 원칙**: 감사에서 "보존" 판정된 페어링은 전부 유지. 교체·조정은 각 테마 브리프 §6에 명시된 항목만(예: 03·12 디스플레이 한글 폴백 Nanum Myeongjo 800 승격, 09 `--font-sans` Noto Sans KR 배선 수리, 13 대형 한글 디스플레이 weight 400 고정). **브리프에 없는 페어링 교체는 DNA 훼손으로 간주.**

**9) 국·영문 혼용 관습**: 조사는 붙여 쓴다("API를", "GitHub에서"). 단어 중간 개행이 치명적인 토큰(`AquaPod Pro`, `₩1,284,000`)은 `.nowrap { white-space: nowrap; }`.

- **수치는 어중간하게**: `12,000`·`50%`·`100+` 반올림은 지문 — `11,842`·`47.3%`·`₩4.82억`처럼 측정된 듯한 값. 시계열은 등락·이상치·계절성의 서사. 합계=부분합, 증감률=실제 계산값 — 대충 넣은 %는 프로가 1초 만에 알아본다.

**3) 마이크로카피 표준**:
- 버튼: 행위 라벨("저장", "초대 보내기"). "확인/취소" 대신 행위 재진술 — "삭제할게요 / 남겨둘게요"(캐주얼) 또는 "삭제 / 취소"(업무).
- 토스트: 결과+후속 — "재고 알림을 켰어요. 입고되면 바로 알려드릴게요." / "메일 3건을 보관했습니다 · 되돌리기".
- 빈 상태·404: 테마 세계관으로 — "주소가 바뀌었거나 점토 속으로 쏙 들어가 버린 것 같아요"(06), "테라조 틈에 빠진 페이지예요"(10).
- 에러: 원인+다음 행동 — "카드 한도가 초과됐어요. 다른 결제 수단을 선택해 주세요."
- placeholder: 형식 예시 — `이름 (예: 김서연)`, `010-0000-0000`.

**4) 번역 금지, 번안 원칙** — 원문의 위트 밀도를 이식하되 직역하지 않는다:

**6) 카피-조판 결합 규칙**:
- **초대형 한글 디스플레이는 카피를 먼저 쓴다** — 2~6자 어절로 줄이 떨어지게 설계("부수고. 만들고. 출시하라." = 4·4·5자). 조판과 카피가 동시에 태어나야 한 방이 된다.
- **한글 자체를 시각 자산으로** — 한글화는 결손 보수가 아니라 증폭 기회: Black Han Sans 초대형 한글, 한글 의성어 버스트, 받침의 기하학·모아쓰기 밀도 등 라틴이 못 하는 조형을 밀어붙일 것.
- 숫자 히어로: 표기 규격+tabular+테마 디스플레이 폰트 3박자 — "₩4.82억"이 초대형 타이포와 만나면 그 자체가 레이아웃이 된다.
- 한글 웹폰트는 라틴보다 획이 가늘어 다크 모드에서 얇아 보인다 — 다크에서 본문 대비 재검.

## 6. 페이지 경험 지침 — 진열대가 아니라 실제 서비스 장면

디자인시스템 데모의 함정은 컴포넌트를 무맥락으로 나열한 '진열대 페이지'다. 각 페이지는 **그 테마의 세계관 속 실제 서비스의 한 장면**이어야 한다.

### 6-1. 페이지 = 장면 (한 문장 시나리오 의무)

각 페이지에 "누가, 무엇을 하는 장면인가"를 한 문장으로 정의하고 그 시나리오가 데이터·카피·상태를 결정하게 한다.

| 페이지 | 진열대 (금지) | 장면 (지향) |
|---|---|---|
| dashboard | KPI 카드 4개+차트 나열 | "6월 마감 주간, 마케터가 여름 캠페인 성과를 확인하는 아침" — 수치에 등락 서사, 이상치 1개, 그에 대한 알림 |
| pricing | 3단 플랜 테이블 | 추천 플랜에 후광(씬 연출), 실결제 맥락(연간 할인 계산값, 국내 결제 수단), FAQ는 실제 궁금할 질문 |
| settings | 폼 컨트롤 총집합 | 저장 실패 토스트 1회, invalid 필드 1개, 위험 구역(계정 삭제)의 확인 플로우 — 상태 시연(§4-3)의 자연 무대 |
| inbox/list | 행 20개 복붙 | 읽음/안읽음 혼재, 고정 항목, 빈 폴더 1개(empty state 시연), 모바일 목록↔읽기 전환 |
| onboarding | step 인디케이터 진열 | 필수값 에러 1회 경험, 완료 순간 = 시그니처 모먼트 후보 |
| 404 | "페이지 없음" 텍스트 | 테마 세계관 최대 강도의 무대(§3-D 클라이맥스 배치) |

### 6-2. 씬 변주 시스템 — 같은 엔진, 다른 연출

배경·분위기 장치를 전 페이지 복붙하지 않는다. `data-scene` 속성 또는 페이지 스코프 토큰으로 페이지 성격에 맞게 변주: 대시보드=데이터가 주인공인 절제 씬, 프라이싱=추천 플랜 뒤 후광, 404=무너진/기울어진 씬. §3-B의 노브(`--fx-*`, `--grain-opacity`)가 그 기반이다.

### 6-3. 상태는 장면 속에서 시연한다

`.is-invalid`·`.cmdk-empty`·skeleton이 CSS에 정의만 되고 화면 노출 0회면 미완성(§4-3). 상태 진열 섹션을 따로 만들지 말고 **장면의 사건으로** 노출한다: settings 저장 실패, onboarding 필수값 에러, 검색 결과 0건, 재고 소진 상품.

### 6-4. 컴포넌트 갤러리(index)의 위계

허브 페이지의 갤러리는 정당한 목적 — 단 §2-3 규칙대로 위계를 만든다: 대표 컴포넌트 히어로 셀, 그리드 침범 요소 1개+, 테마 어법의 갤러리 재해석(목차/등록 대장/도록/스페시멘).

### 6-5. 페이지 간 밀도 균일화

세계관 밀도가 강한 페이지(대개 index·dashboard)와 약한 페이지(pricing·settings·onboarding)의 낙차를 없앤다. 약한 페이지를 강한 페이지 수준으로 — 특히 테크·터미널형(§3-E 유형 4)에서 로그 픽션·명판 밀도를 전 페이지 균일하게.

## 7. 구현 체크리스트 (테마당, 작업 마지막 자가 검증)

### 7-0. 작업 순서 템플릿

1. **P0 위생**: SVG var() 전수 교체 → 전면 한글화(pages+app.js) → 폰트 스택 순서/죽은 임포트 → FOUC 스니펫 → 죽은 컨트롤(버거/닫기) → 인라인 그리드 승격

3. **P2 탈지문**: 자기참조 스탯·raw .md 링크·이모지·filler 데이터·복붙 장식 제거(§2)
4. **P3 대담화**: 분위기 점화(죽은 토큰 회수) → 히어로 재조판 → 시그니처 모션 교체 → 시그니처 모먼트 1개 완성(§3) → 페이지 장면화(§6)
5. **검증**: 아래 7-1 + 7-2

### 7-1. 자동 검증 (grep — 결과를 CHECKLIST.md에 기록)

```bash
T=design-systems/theme-XX-*/

# 1. SVG presentation attr var() — 0건
grep -rn 'stroke="var(\|fill="var(\|stop-color="var(\|color="var(\|font-family="var(' --include="*.html" $T
# 2. lang — 전 HTML lang="ko" (미포함 파일 출력 0)
grep -L 'lang="ko"' $T/*.html $T/pages/*.html
# 3. 하드코딩 hex (components 레이어) — intentional 주석 제외 0건
grep -rn '#[0-9a-fA-F]\{3,6\}' $T/components/*.css | grep -v 'intentional'
# 4. 폰트: 임포트 목록 vs 스택 대조 (미사용 패밀리 0) + Noto Serif KR 잔존 여부
grep -rn '@import\|fonts.googleapis' $T/*.css $T/*.html
# 5. raw .md 링크 / href="#" / 인라인 onclick — 0건
grep -rn 'href="[^"]*\.md"\|href="#"\|onclick=' --include="*.html" $T
# 6. 인라인 grid-template-columns — 0건
grep -rn 'style="[^"]*grid-template-columns' --include="*.html" $T
# 7. 무효 ARIA — plain button aria-selected (role="tab|option|row" 문맥만 허용)
grep -rn 'aria-selected' --include="*.html" $T
# 8. 미정의 토큰 참조 — 토큰 정의 목록과 diff
grep -roh 'var(--[a-z0-9-]*' $T | sort -u
# 9. 서구권 filler 잔재 — 0건
grep -rn '\$[0-9]\|€\|PM<\|ago<\|Trusted by\|Acme\|Lorem\|>Claude<' --include="*.html" $T
# 10. 한글 조판 계약 — keep-all 존재 확인
grep -rn 'keep-all' $T/*.css
```
