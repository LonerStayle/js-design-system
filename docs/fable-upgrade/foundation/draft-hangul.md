# PART A. 한글 조판 기저 규격

## 1. 줄바꿈 계약 — keep-all은 전역, 안전망은 break-word

**현황**: 30개 테마 전수에서 `word-break: keep-all` 사용 0건(grep 검증). 한글화가 진행될수록 어절 중간이 뚝뚝 끊기는 "AI 번역기 조판"이 그대로 노출된다. 가장 값싸고 효과 큰 수리.

```css
/* base.css — 전역 계약 */
body {
  word-break: keep-all;          /* 한글 어절 단위 줄바꿈 */
  overflow-wrap: break-word;     /* 긴 URL·영문 토큰 오버플로 안전망 */
  line-break: strict;            /* 조사·괄호 등 금칙 처리 엄격화 */
}

/* 헤딩·카드 타이틀 — 두 줄 균형 (지원 브라우저 한정 향상) */
h1, h2, h3, .card__title, .hero__title { text-wrap: balance; }
/* 본문 문단 — 외톨이 낱말 방지 */
p { text-wrap: pretty; }
```

**함정 3건 — 반드시 함께 처리**:

1. `keep-all`은 좁은 컨테이너(테이블 셀, 칩, 배지, 사이드바)에서 **한 어절이 컨테이너보다 길면 오버플로**한다. `overflow-wrap: break-word`가 안전망이지만, 셀·칩류는 애초에 말줄임이 정답:
```css
.table td, .chip, .badge, .tab__label {
  white-space: nowrap; overflow: hidden; text-overflow: ellipsis;
}
```
2. 기존 `:where(p, li, figcaption) { overflow-wrap: break-word; }` (theme-01 base.css:59 등)는 유지하되 body 전역 선언과 중복이므로 정리 가능.
3. 초대형 디스플레이 타이포(§5 대담화의 뷰포트급 헤드라인)는 keep-all 상태에서 어절이 통째로 내려가 행 균형이 깨질 수 있다 → 카피 자체를 **의도한 줄 단위로 `<br>` 또는 `<span>` 분절**해서 조판하라. 프로 포스터는 줄바꿈을 브라우저에 맡기지 않는다.

```html
<!-- 히어로 헤드라인은 줄을 '설계'한다 -->
<h1 class="display">
  <span>부수고. 만들고.</span>
  <span>출시하라.</span>
</h1>
```

## 2. 행간 — 한글은 라틴보다 한 단계 넉넉하게, 디스플레이는 이원화

한글은 어센더/디센더 없이 글자 상자가 꽉 차는 문자라 같은 수치의 행간에서 라틴보다 답답해 보인다. 현행 토큰 램프(`--leading-none:1 / tight:1.1 / snug:1.25 / normal:1.5 / relaxed:1.7`)는 유지하되 **역할 배정을 한글 기준으로 재계약**한다.

| 역할 | 라틴 전용일 때 | 한글 포함 시 기저 | 근거 |
|---|---|---|---|
| 본문 단락 | 1.5~1.6 | **1.65~1.75** (`--leading-relaxed`) | 현행 `p { line-height: var(--leading-relaxed) }` 유지 — 이미 적정 |
| UI 텍스트(버튼·라벨·셀) | 1.2~1.4 | **1.4~1.5** | 받침 시각 균형 |
| 헤딩(다행) | 1.1~1.2 | **1.25~1.35** (`--leading-snug`) | 현행 `--leading-tight:1.1`을 한글 다행 헤딩에 쓰면 받침·아래 획이 시각적으로 붙음 |
| 디스플레이(초대형) | 0.9~1.05 | **1.05~1.2** | `line-height: 0.95` (theme-01 `.display`)를 한글에 그대로 쓰면 겹침·잘림 |

**구현 패턴 — 토큰 이원화 + :lang 분기** (전 테마 공통 이식):

```css
:root {
  --leading-display: 0.95;        /* 라틴 기준 (테마 기존값 유지) */
  --leading-display-ko: 1.12;     /* 한글 디스플레이 보정값 */
  --leading-heading-ko: 1.3;
}

.display { line-height: var(--leading-display); }

/* 한글 콘텐츠 보정 레이어 — lang="ko" 전환(§5-1)이 선행 조건 */
:lang(ko) .display,
.display:lang(ko) { line-height: var(--leading-display-ko); }
:lang(ko) :is(h1, h2, h3) { line-height: var(--leading-heading-ko); }
```

단일 언어 페이지(전면 한글화 후)는 `html lang="ko"` 하나로 전체 보정이 걸리므로 위 셀렉터가 그대로 동작한다. 한 요소 안에 국·영문이 섞일 때는 큰 쪽(한글) 기준으로 잡는다 — 라틴이 손해 보는 방향이 항상 안전하다.

`--leading-none(1.0)`은 **한 줄짜리 한글 워드마크·스탯 숫자에만 허용**, 다행 한글에 사용 금지.

## 3. 자간 — 라틴 트래킹을 한글에 이월하지 말 것

현행 트래킹 램프(-0.04em ~ 0.12em, 테마별 caps 0.34em까지)는 전부 라틴 기준 설계다. 감사에서 실제 적발된 사례: theme-12 아르데코 `--tracking-caps: 0.34em`이 한글 디스플레이에 그대로 적용(브리프 §6 "한글에는 0.2em 안팎 별도 보정 권장"), theme-02 모노 라벨 tracking-widest의 한글 과확장 경고.

**기저 수치 계약**:

| 상황 | 라틴 | 한글 기저 |
|---|---|---|
| 본문 400~500 (고딕 계열) | 0 | **-0.01em ~ 0** — Noto/Gothic A1은 살짝 좁혀야 현대적 |
| 본문 (명조 계열: 나눔명조·고운바탕) | 0 | **0** — 명조는 좁히면 획이 엉킴 |
| 디스플레이 48px+ (일반 고딕) | -0.02 ~ -0.04em | **-0.02em** 상한 |
| 디스플레이 (자체 고밀도 무드폰트: Black Han Sans·Do Hyeon·Jua) | — | **0** — 이미 콘덴스드, 더 좁히면 뭉개짐 |
| 와이드 라벨(라틴 uppercase 0.1~0.34em) | 그대로 | **0.05~0.1em**, 아르데코급 극단 트래킹도 한글은 **0.15~0.2em 상한** |

```css
:root {
  --tracking-caps: 0.34em;        /* 라틴 uppercase 라벨 (테마 기존값) */
  --tracking-caps-ko: 0.18em;     /* 같은 위계의 한글 라벨 */
}
.eyebrow { letter-spacing: var(--tracking-caps); }
:lang(ko) .eyebrow { letter-spacing: var(--tracking-caps-ko); }
```

**주의**: 한글 마지막 글자 뒤에도 자간이 붙어 중앙정렬이 오른쪽으로 밀려 보인다. 넓은 자간의 중앙정렬 한글 라벨에는 `margin-right: calc(var(--tracking-caps-ko) * -1)` 또는 좌측 `padding-left`로 시각 보정.

## 4. 국·영문 혼용 리듬

### 4-1. 폰트 스택 순서 — 이번 감사 최다 적발 결함 (P1)

per-glyph fallback 원리: 라틴 글리프는 앞의 라틴 폰트가 선점하고, 한글 글리프만 뒤로 흘러 한글 폰트에 도달한다. 그런데 `system-ui`/`-apple-system`/`Segoe UI`가 한글 웹폰트보다 **앞**에 있으면 macOS에선 Apple SD Gothic Neo가 한글을 가로채 무드 폰트가 죽는다 (theme-01·02·04·05·10·16·18·21에서 적발 — "의도한 페어링이 실제로는 미적용" 상태).

**표준 스택 계약** (전 테마 강제):

```css
/* 순서: 라틴 무드 → 한글 무드 → 한글 안전망 → 시스템 → generic */
--font-display: "Space Grotesk", "Archivo",            /* ① 라틴 */
                "Black Han Sans",                      /* ② 한글 무드 */
                "Noto Sans KR",                        /* ③ 한글 안전망 */
                system-ui, -apple-system, sans-serif;  /* ④ 최후 폴백 */
```

이 순서에서 라틴·숫자는 여전히 ①이 선점하므로 부작용이 없다. **숫자가 라틴 폰트로 렌더되는 것 자체가 국·영문 혼용 리듬의 뼈대다** — 한글 폰트의 숫자보다 라틴 전용 폰트의 숫자가 대체로 미려하고, tabular 피처(§4-4)도 라틴 폰트에서 온다.

### 4-2. 이탤릭 금지 — 한글엔 이탤릭이 없다

`<em>`·`.binomial`·인용 이탤릭이 한글에 걸리면 faux-oblique(기계 기울임)로 왜곡된다 (theme-15 `<em>집중</em>`, theme-17 학명 이탤릭에서 적발).

```css
:lang(ko) em, :lang(ko) i, :lang(ko) cite {
  font-style: normal;
  font-weight: var(--weight-semibold);   /* 굵기 강조로 치환 */
  color: var(--color-accent);            /* 또는 테마 강조색 — 택1/병용은 테마 보이스 따라 */
}
```

라틴 학명·인용구는 라틴 문자로 유지해 이탤릭을 살린다(theme-17 판정). 명조 테마에서는 이탤릭 대신 **서체 전환**(본문 나눔명조 → 강조 고운바탕)으로 목소리를 구분하는 것도 브리프 검증된 대안.

### 4-3. 스몰캡스·uppercase 무효 — 한글 라벨 보정 규칙

`font-variant: small-caps`·`text-transform: uppercase`는 한글에 아무 효과가 없다. 라틴 라벨과 같은 위계로 보이게 하려면 **크기 축소 + 자간 + (테마에 따라) 서체 전환**의 3종 세트로 치환한다 (theme-17 브리프 처방 채택):

```css
.label { font-variant: small-caps; letter-spacing: var(--tracking-widest); }

:lang(ko) .label {
  font-variant: normal;
  font-size: var(--text-xs);              /* 라틴 스몰캡스와 시각 높이 맞춤 */
  letter-spacing: var(--tracking-caps-ko);/* §3 상한 준수 */
  font-weight: var(--weight-medium);
}
```

### 4-4. 숫자 피처 — tabular은 데이터에, 비례는 본문에

```css
/* 테이블·스탯·카운터·가격 — 자릿수 정렬 */
.table td, .stat__value, .price, .countdown, [data-numeric] {
  font-variant-numeric: tabular-nums lining-nums;
}
/* (구형 표기 병행 시) font-feature-settings: "tnum" 1, "lnum" 1; */
```

theme-03 에디토리얼의 old-style(`"onum"`) 본문 + lining/tabular 스탯 전환은 이 규칙의 모범 — 인쇄 DNA 테마는 그 관습을 유지·확산하고, 나머지 테마는 tabular/lining을 기본으로 한다. **주의**: 한글 폰트로 렌더되는 숫자에는 tnum이 없는 경우가 많다 — 숫자가 라틴 폰트에서 렌더되도록 스택 순서(§4-1)가 선행 조건.

### 4-5. 한·영 문장 리듬 (마크업 관습)

- 한글 문장 속 영단어·숫자 앞뒤에 공백을 강제 삽입하지 않는다. 조사는 붙여 쓴다: "API를", "GitHub에서".
- 영문 고유명사는 첫 등장만 원문, 이후 한글 표기 가능 — 카피 보이스(§7)의 재량.
- 단어 중간 개행이 치명적인 토큰(모델명 `AquaPod Pro`, 금액 `₩1,284,000`)은 `<span class="nowrap">` 처리:
```css
.nowrap { white-space: nowrap; }
```

## 5. 전면 한글화 계약 (조판 전제 조건)

23개 브리프 중 **한글화 미완이 최다 P0** (01·02·08·10·11·15·17·21·22·23 등 — "pages 9개 전부 영문, index만 한글"). 조판 스펙은 한글화가 끝나야 실동작한다.

1. `<html lang="ko">` — 전 페이지. 스크린리더 발음 규칙에 직결.
2. 한글화 범위: 보이는 텍스트 + `aria-label`/`alt`/`title`/`placeholder` + **app.js 내장 문자열**(토스트 기본 타이틀, "Go to slide", "Notifications" 등) + 인라인 `onclick` JS 문자열.
3. **스크린리더 문구는 테마 DNA와 무관하게 무조건 한국어** — theme-07 사이버펑크처럼 표면 영문이 DNA인 테마도 "알림 닫기"·"이전 달"은 한국어 (브리프 §3-D 확정).
4. 영문 유지: 코드 블록, 클래스명, 기술 약어, 워드마크, DNA 장식 텍스트(§0 표).
5. 데이터 현지화: 인명 한국식, 통화 ₩, 날짜 한국 형식(§6) — 번역이 아니라 **번안**(§7-5).

## 8. 공통 배선 수리 4종 (전 테마 일괄)

1. **스택 순서 교정** — §4-1 계약대로. 라틴 → 무드한글 → Noto KR → system-ui.
2. **죽은 Noto Serif KR import 전량 제거** — 세리프가 DNA인 테마(03·12·13·15·17·18, 잔여 브리프에서 추가 확인)를 **제외한** 전 테마에서 "임포트만 있고 스택 미사용" 화석으로 적발됨(01·02·04·05·06·07·09·10·11·14·16·19·20·21·22·23). 렌더 블로킹 페이로드 + 서재 관성 흔적.
3. **웨이트 정합 — faux-bold 근절**:
   - 헤딩이 쓰는 웨이트(700/800/900)를 한글 폰트가 실제로 보유·로드하는지 대조. 미보유 시: (a) 보유 폰트로 폴백 승격(§7 표), (b) 웨이트 추가 로드(Gothic A1 800 등), (c) 단일 웨이트 무드폰트(Black Han Sans·Do Hyeon·Jua 400)는 **한글 헤딩 weight 분기**:
   ```css
   /* 단일 웨이트 무드폰트 테마 — 합성 볼드 차단 + 위계는 크기·자간으로 */
   :lang(ko) :is(h1, h2, .display) {
     font-synthesis-weight: none;   /* 지원 브라우저 */
     font-weight: 400;              /* 폰트 실웨이트에 고정 */
   }
   ```
4. **로딩 단일화** — `@import` 2회 분산(tokens.css + base.css) 테마 다수 → 한 곳 통합, 가능하면 HTML `<link rel="preload">`/`<link>` 승격 + `display=swap` 유지.

## 9. `:lang(ko)` 한글 보정 레이어 — 표준 이식 스니펫

각 테마 base.css 하단에 붙이는 공통 블록. **테마 토큰만 참조**하고 하드코딩 없음(전 테마의 토큰 규율 준수).

```css
/* ============================================
   Hangul refinement layer (기저 스펙 §9)
   전제: <html lang="ko">, 폰트 스택 순서 §4-1 교정 완료
   ============================================ */
:lang(ko) {
  word-break: keep-all;
  overflow-wrap: break-word;
  line-break: strict;
}

/* 행간 이원화 — 값은 테마 토큰으로 등록 후 참조 */
:lang(ko) :is(h1, h2, h3) { line-height: var(--leading-heading-ko, 1.3); }
:lang(ko) .display        { line-height: var(--leading-display-ko, 1.12); }

/* 자간 보정 */
:lang(ko) :is(.eyebrow, .label, .btn--caps) {
  letter-spacing: var(--tracking-caps-ko, 0.08em);
  text-transform: none;           /* 한글에 무효 — 명시적으로 끔 */
  font-variant: normal;
}

/* 이탤릭 치환 */
:lang(ko) :is(em, i, cite) {
  font-style: normal;
  font-weight: var(--weight-semibold, 600);
}

/* 데이터 숫자 정렬 (라틴 폰트 선점이 전제) */
:lang(ko) :is(.table td, .stat__value, .price) {
  font-variant-numeric: tabular-nums lining-nums;
}

/* 단일 웨이트 무드폰트 테마에서만 활성화 (§8-3) */
/* :lang(ko) :is(h1, h2, .display) { font-synthesis-weight: none; font-weight: 400; } */
```

주의: 이 레이어는 **기저값**이다. 인쇄 DNA 테마(03 old-style 숫자 등)는 자기 조판 관습이 이 레이어를 오버라이드한다 — §0 우선순위.

## 11. 실감 카피 작성 규격 — "실제 한국 서비스에 있을 법한"

### 11-1. 인명
- 한국 성명 2~4자, 세대·직군 실감 분포: 김서연, 박지훈, 이도윤, 한예리, 서지원, 최민준, 정하은, 오세훈 대리, 윤 팀장.
- 같은 화면 안 성씨 중복 회피(전원 김씨 금지), 직함·역할 병기로 실재감: "편집장 서지원", "문화부 한예리".
- 외국인 1명 정도 허용(국제 협업 실감) — 단 테마 세계관 정합 시에만. theme-10 멤피스의 실존 디자이너 페르소나(엣토레 소트사스)처럼 **DNA 지식이 배어든 인명은 한글 표기로 유지**.

### 11-2. 회사·상품·캠페인명
- 국내 서비스 작명 관습: "온새미로 스튜디오", "달빛서점", "모두의가구", "핀치 트래블".
- 캠페인: "여름 신규가입 캠페인", "6월 멤버십 리텐션", "추석 선물세트 사전예약".
- 상품: 카테고리+속성 실감 조합 — "피오르 오크 사이드 테이블", "저소음 무접점 키보드 K3".

### 11-3. 수치 — 어중간한 숫자가 실감을 만든다
- `12,000` `50%` `100+` 같은 반올림 숫자는 AI 지문. **`11,842` `47.3%` `₩4.82억`** 처럼 측정된 듯한 값.
- 시계열은 서사가 있어야 한다: theme-09의 검증된 모범 — 차트 aria-label "38백만원에서 시작해 … 6월 82백만원으로 상승". 등락·이상치·계절성을 심을 것.
- 수치 간 정합: 합계=부분합, 증감률=실제 계산값. 대충 넣은 %는 프로가 1초 만에 알아본다.

### 11-4. 마이크로카피 표준
- **버튼**: 동사로 끝나는 행위 라벨 — "저장", "내보내기", "초대 보내기". "확인/취소" 쌍 대신 행위 재진술: "삭제할게요 / 남겨둘게요"(캐주얼 테마) 또는 "삭제 / 취소"(업무 테마).
- **토스트**: 결과+후속 — "재고 알림을 켰어요. 입고되면 바로 알려드릴게요." / 실행취소 제공 시 "메일 3건을 보관했습니다 · 되돌리기".
- **빈 상태**: 테마 세계관으로. 검증된 모범: "주소가 바뀌었거나 점토 속으로 쏙 들어가 버린 것 같아요"(06), "새로운 소식이 도착하면 여기 별처럼 반짝일 거예요"(18), "테라조 틈에 빠진 페이지예요"(10 번안 지침).
- **에러**: 원인+다음 행동 — "카드 한도가 초과됐어요. 다른 결제 수단을 선택해 주세요."
- **placeholder**: 형식 예시를 담는다 — `이름 (예: 김서연)`, `010-0000-0000`.
- **개인정보·보안 카피**: 안심 문장은 구체적으로 — 검증된 모범 "절대 공유하지 않습니다. 약속해요."(01) 톤을 pages까지 확장.

## 13. 카피-조판 결합 규칙 (이 문서 고유의 접점)

1. **초대형 한글 디스플레이는 카피를 먼저 쓴다** — 2~6자 어절로 줄이 떨어지게 설계("부수고. 만들고. 출시하라." = 4·4·5자). 조판(§1-3)과 카피가 동시에 태어나야 한 방이 된다.
2. **한글 자체를 시각 자산으로** — 한글화는 결손 보수가 아니라 증폭 기회다: Black Han Sans 초대형 한글(01), 한글 의성어 버스트(22 `BURST_WORDS` → "쾅!·펑!·빠밤!"), Gothic A1 800 두 줄 스택 포스터(02). 라틴이 못 하는 조형(받침의 기하학, 모아쓰기 밀도)을 밀어붙일 것.
3. **숫자 히어로**: 대시보드 KPI·가격은 카피 규격(§6)+tabular(§4-4)+테마 디스플레이 폰트의 3박자. "₩4.82억"처럼 축약 표기가 초대형 타이포와 만나면 그 자체가 레이아웃이 된다.
4. **다크·라이트 모두에서 검수**: 한글 웹폰트는 라틴보다 획이 가늘어 다크 모드에서 한 단계 desaturate/얇아 보임 — 다크에서 본문 대비 재검(각 테마 CHECKLIST 실측 관습 유지).
